## 推荐默认方案

如果你想先快速推进，可以直接采用这套默认值：

1. 目标：提取文本 + 结构化字段 + must-have / nice-to-have 分类
2. PDF 类型：第一版只支持文本型 PDF
3. 输出：原始文本 + 结构化 JSON + 不确定项提示
4. 字段：岗位名称、摘要、must-have、nice-to-have、年限、行业、学历、项目关键词、技术栈
5. 推断：允许弱推断，但必须标记不确定
6. 失败处理：返回部分结果 + 风险提示
7. 用户编辑：允许
8. 保存：至少保存提取文本和结构化结果
9. 架构：独立 JD PDF Agent，输出接入现有匹配流程
10. 日志：沿用现有运行日志，增加 JD 解析事件
11. 交互：上传后解析，用户确认后进入匹配
12. 验收：提取稳定、字段可用、可修正、可接入匹配
